# Tekka API一覧

## 1. 概要

本資料では、TekkaのPhase 1で使用するAPIの一覧を定義する。

各APIのリクエスト項目、レスポンス項目、バリデーション、ステータスコード、認可、処理内容などの詳細は、分類ごとのAPI仕様書で定義する。

API全体の共通ルールについては、`api-common.md`を参照する。

---

## 2. 認証API

詳細仕様は、`auth-api.md`で定義する。

| メソッド | URL                      | 認証 | 概要                             |
| -------- | ------------------------ | ---- | -------------------------------- |
| `GET`  | `/sanctum/csrf-cookie` | 不要 | SanctumのCSRF Cookieを取得する   |
| `POST` | `/api/register`        | 不要 | ユーザーを新規登録する           |
| `POST` | `/api/login`           | 不要 | ログインする                     |
| `POST` | `/api/logout`          | 必要 | ログアウトする                   |
| `GET`  | `/api/user`            | 必要 | ログインユーザーの情報を取得する |

`GET /sanctum/csrf-cookie`は、Tekka独自のAPIではなく、Laravel Sanctumが提供するエンドポイントである。

---

## 3. ユーザーAPI

詳細仕様は、`users-api.md`で定義する。

| メソッド | URL                          | 認証 | 概要                                         |
| -------- | ---------------------------- | ---- | -------------------------------------------- |
| `GET`  | `/api/users/{id}`          | 不要 | 指定ユーザーの公開プロフィールを取得する     |
| `PUT`  | `/api/user`                | 必要 | ログインユーザー自身のプロフィールを更新する |
| `GET`  | `/api/users/{id}/articles` | 不要 | 指定ユーザーが投稿した記事一覧を取得する     |

`GET /api/user`は認証APIとして扱い、ユーザーAPIには重複して記載しない。

---

## 4. 記事API

詳細仕様は、`articles-api.md`で定義する。

| メソッド   | URL                    | 認証 | 概要                                     |
| ---------- | ---------------------- | ---- | ---------------------------------------- |
| `GET`    | `/api/articles`      | 不要 | 記事一覧を取得する                       |
| `GET`    | `/api/articles/{id}` | 不要 | 指定記事の詳細を取得する                 |
| `POST`   | `/api/articles`      | 必要 | 新しい記事を投稿する                     |
| `PUT`    | `/api/articles/{id}` | 必要 | ログインユーザー自身の記事を更新する     |
| `DELETE` | `/api/articles/{id}` | 必要 | ログインユーザー自身の記事を論理削除する |

記事編集画面専用の取得APIは作成せず、記事詳細取得APIを使用する。

```text
GET /api/articles/{id}
```

---

## 5. タグAPI

詳細仕様は、`tags-api.md`で定義する。

| メソッド | URL                               | 認証 | 概要                               |
| -------- | --------------------------------- | ---- | ---------------------------------- |
| `GET`  | `/api/tag-articles` | 不要 | 指定タグが付いた記事一覧を取得する |

取得対象のタグ名は、`tag` クエリパラメータで指定する。

```text
GET /api/tag-articles?tag=laravel&page=1
```

タグは記事投稿・編集時に自動作成または再利用するため、タグ単体の作成・更新・削除APIは作成しない。

Phase 1では、タグ一覧取得APIおよびタグ入力のオートコンプリートAPIも作成しない。

---

## 6. 検索API

詳細仕様は、`search-api.md`で定義する。

| メソッド | URL             | 認証 | 概要                               |
| -------- | --------------- | ---- | ---------------------------------- |
| `GET`  | `/api/search` | 不要 | キーワードを使用して記事を検索する |

検索キーワードは、`q`クエリパラメータで指定する。

```text
GET /api/search?q=laravel&page=1
```

検索対象や空の検索キーワードの扱いなどは、検索API仕様書で定義する。

---

## 7. 評価API

詳細仕様は、`evaluations-api.md`で定義する。

| メソッド | URL                               | 認証 | 概要                                       |
| -------- | --------------------------------- | ---- | ------------------------------------------ |
| `POST` | `/api/articles/{id}/evaluation` | 必要 | 指定記事への評価を作成・切り替え・取り消す |

評価APIは、送信された評価値と現在の評価状態に応じて、次のいずれかを行う。

- 評価の作成
- 高評価と低評価の切り替え
- 同じ評価を再度選択した場合の取り消し

評価一覧取得APIおよび評価者一覧取得APIは作成しない。

---

## 8. API件数

Phase 1で使用するエンドポイント数は、次のとおりである。

| 分類           |         件数 |
| -------------- | -----------: |
| 認証API        |            5 |
| ユーザーAPI    |            3 |
| 記事API        |            5 |
| タグAPI        |            1 |
| 検索API        |            1 |
| 評価API        |            1 |
| **合計** | **16** |

このうち、`GET /sanctum/csrf-cookie`はLaravel Sanctumが提供するエンドポイントである。

Tekkaが独自に定義するAPIは15件である。
